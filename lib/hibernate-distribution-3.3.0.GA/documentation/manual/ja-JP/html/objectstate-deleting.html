10.8. 永続オブジェクトの削除

Session.delete() はオブジェクトの状態をデータベースから削除します。 もちろん、削除したオブジェクトをアプリケーションが保持したままでもよいです。 そのため、 delete() は永続インスタンスを一時的にするものと考えるのが一番です。 

sess.delete(cat);

外部キー制約に違反するリスクもなく、好きな順番でオブジェクトを削除することができます。 ただし、間違った順番でオブジェクトを削除すると、外部キーカラムの NOT NULL 制約に違反する可能性があります。 例えば、親オブジェクトを削除したときに、子供オブジェクトを削除し忘れた場合です。
